### 機能の目的

会員詳細画面（A-01-01）のタブ切り替え領域に「支払い履歴」タブを追加し、対象会員の入出金明細・請求一覧・支払いサマリーを一覧参照できるようにする。スタッフが会員対応中にその場で料金状況を確認できることで、業務効率と対応品質を向上させる。

## Clarifications

### Session 2026-04-22

- Q: What does `currentMonthAmount` (今月請求額) include — monthly fee only, or all billing types for the current month? → A: Sum of **all** billing records (monthly + one-time) for the current calendar month, regardless of status.
- Q: How are the four billing statuses (`未確定` / `入金済み` / `未回収` / `貸倒`) defined and do they transition? → A: `未確定` = not yet invoiced/confirmed; `入金済み` = payment received; `未回収` = overdue but still collectible; `貸倒` = written off as unrecoverable (distinct and non-transitive at the display layer).
- Q: How should the transaction ledger handle large history datasets? → A: Pagination (50 rows per page) using the existing `TablePagination` component; no lazy-load.
- Q: For option-only members, should the Payment History tab be accessible? → A: **Always visible and fully accessible** (no row/record filtering). Other tabs may be restricted, but Payment History remains enabled for all members.
- Q: How should API errors be handled and retried? → A: Three separate API calls (ledger, billing list, summary); React Query manages caching/retries; show error message + manual retry button on failure.
- Q: What are the explicit acceptance criteria for this feature? → A: (1) Tab renders & loads within 3s per call, (2) pagination works (next/prev/page numbers), (3) filters (period/type) correctly narrow results, (4) status badges display correct colors, (5) option-only members can access without 403, (6) error state shows with manual retry button.
- Q: Should UI labels follow a specific language/style (Japanese vs. English vs. bilingual)? → A: **Japanese labels from UI prototype are the source of truth** for all user-facing text; internal/technical naming (components, APIs) may use English.

## 機能要件

### FR-009-a 入出金明細の表示

- 対象会員の売上・返金の全履歴を一覧表示する
- 表示項目: **日付 / 種別 / 内容 / 金額 / 決済方法**
- 日付降順で表示する
- **フィルタリング**（画面上部）:
  - 期間: `全期間` / `今月` / `先月` / `過去3ヶ月` / `過去6ヶ月`
  - 種別: `全種別` / `売上` / `返金`
- 金額表示:
  - 売上（正値）: `¥9,900` 形式
  - 返金（負値）: `-¥2,200` 形式でdestructive色表示
- 種別は Badge で表示: 返金 → secondary variant、売上 → default variant

### FR-009-b 請求一覧の表示

- 月次請求・都度請求の一覧を表示する
- 表示項目: **請求月 / 請求種別 / 金額 / ステータス / 請求日**
- 請求ステータスの種類と定義:

  | ステータス | 定義                                               | 表示色              |
  | ---------- | -------------------------------------------------- | ------------------- |
  | 未確定     | 未だ確定・請求されていない状態                     | info（青系）        |
  | 入金済み   | 代金が入金/回収済み                                | success（緑系）     |
  | 未回収     | 請求後、支払期限を超過したが回収を継続している状態 | warning（橙系）     |
  | 貸倒       | 回収不能と判定され、会計上貸倒処理済みの状態       | destructive（赤系） |

- 請求種別は Badge で表示: 都度 → outline variant、月次 → default variant

### FR-009-c 支払いサマリーの表示

- 右カラム（40%幅）にスティッキー表示する
- 表示項目:

  | 項目       | 説明                                                           |
  | ---------- | -------------------------------------------------------------- |
  | 今月請求額 | 当月の全請求レコードの合算（月次＋都度。ステータス問わず集計） |
  | 未回収合計 | ステータスが「未回収」または「貸倒」の合計金額                 |
  | 最終入金日 | 最後に入金済みとなった日付                                     |
  | 決済方法   | 現在の決済手段（例: SBPS / JACCS / 現金）                      |

### FR-009-d アクセス制御（オプション契約のみ会員）

- **支払い履歴タブは常に表示・アクセス可能**。オプション契約のみ会員であっても、このタブの全機能（入出金明細・請求一覧・サマリー）にアクセス可能
- 他タブ（基本情報、契約操作等）の制限は本スペックの範囲外だが、支払い履歴は制限対象外と明記
- 権限外会員（全く所属がない、など）への直接アクセスは 403 エラーを返す

## データ定義

### 入出金明細 (PaymentHistoryItem)

| フィールド | 型                    | 説明                                           |
| ---------- | --------------------- | ---------------------------------------------- |
| `date`     | `string` (YYYY/MM/DD) | 入出金日                                       |
| `type`     | `"売上" \| "返金"`    | 取引種別                                       |
| `content`  | `string`              | 取引内容（例: 月会費（4月分））                |
| `amount`   | `number`              | 金額（返金は負値）                             |
| `method`   | `string`              | 決済方法（例: SBPS / JACCS / 現金 / 銀行振込） |

### 請求一覧 (BillingItem)

| フィールド    | 型                                             | 説明                    |
| ------------- | ---------------------------------------------- | ----------------------- |
| `month`       | `string`                                       | 請求月（例: 2026年4月） |
| `type`        | `"月次" \| "都度"`                             | 請求種別                |
| `amount`      | `number`                                       | 請求金額（円）          |
| `status`      | `"未確定" \| "入金済み" \| "未回収" \| "貸倒"` | 請求ステータス          |
| `billingDate` | `string` (YYYY/MM/DD)                          | 請求日                  |

### 支払いサマリー (PaymentSummary)

| フィールド           | 型               | 説明                          |
| -------------------- | ---------------- | ----------------------------- |
| `currentMonthAmount` | `number`         | 今月請求額                    |
| `unpaidTotal`        | `number`         | 未回収合計                    |
| `lastPaymentDate`    | `string \| null` | 最終入金日（履歴なしは null） |
| `paymentMethod`      | `string`         | 決済方法                      |

## 状態・表示パターン

| 状態                    | 挙動                                                                                   |
| ----------------------- | -------------------------------------------------------------------------------------- |
| 履歴データが存在する    | 入出金明細・請求一覧を表形式で表示                                                     |
| 入出金明細が0件         | 「入出金履歴はありません」のエンプティステートを表示                                   |
| 請求一覧が0件           | 「請求履歴はありません」のエンプティステートを表示                                     |
| フィルタ結果が0件       | 「該当する履歴はありません」を表示                                                     |
| 未回収合計が0より大きい | サマリーの未回収合計をdestructive色で強調表示                                          |
| APIエラー               | エラーメッセージを表示し、手動リトライボタンを提供。React Query が再試行ロジックを管理 |

## API Integration

このタブは以下の3つの独立した API エンドポイントから データを取得します:

| エンドポイント                                          | 用途                               | キャッシュ戦略                        |
| ------------------------------------------------------- | ---------------------------------- | ------------------------------------- |
| `GET /crm/members/{id}/payment-history?page=1&limit=50` | 入出金明細（ページネーション対応） | React Query のキャッシュ + 自動再試行 |
| `GET /crm/members/{id}/billing?page=1&limit=50`         | 請求一覧（ページネーション対応）   | React Query のキャッシュ + 自動再試行 |
| `GET /crm/members/{id}/payment-summary`                 | 支払いサマリー                     | React Query のキャッシュ              |

- エラー時: エラーメッセージ + 手動リトライボタンを表示。自動リトライは React Query が管理
- タイムアウト: 10秒（要確認）

## 非機能要件

- データは読み取り専用。このタブからの編集・削除操作は提供しない
- **ページネーション**: 入出金明細は1ページ50件で、既存の `TablePagination` コンポーネントを使用。請求一覧も同様

## 制約・ブランド差異

- ブランド間の請求データを横断参照することはできない（会員が所属するブランドの請求のみ表示）
- 決済方法の種類（SBPS / JACCS 等）はシステム設定に依存し、ハードコードしない
- ブランド別の表示差異: なし（表示項目は共通）
